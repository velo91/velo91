# Install Bersih MariaDB 11.4 di Ubuntu 24.04

## Install dependency

```bash
sudo apt install curl gnupg
```

---

## Tambahkan key MariaDB

```bash
sudo mkdir -p /etc/apt/keyrings

curl -fsSL https://mariadb.org/mariadb_release_signing_key.pgp \
| sudo gpg --dearmor -o /etc/apt/keyrings/mariadb-keyring.pgp
```

---

## Tambahkan repo MariaDB 11.4

```bash
sudo nano /etc/apt/sources.list.d/mariadb.list
```

Isi (cara ini tidak support untuk ubuntu 26.04 karena saat ini belum ada metadata 26.04 untuk versi 11.4, yang ada versi 11.8):

```bash
deb [signed-by=/etc/apt/keyrings/mariadb-keyring.pgp] https://mirror.mariadb.org/repo/11.4/ubuntu noble main
```

---

## Update repo

```bash
sudo apt update
```

---

## Install MariaDB 11.4
## kalau ubuntu 26.04, versinya MariaDB 11.8

```bash
sudo apt install mariadb-server mariadb-client
```

---

## Jalankan service

```bash
sudo systemctl enable mariadb && sudo systemctl restart mariadb
```

---

## Cek versi

```bash
mariadb -V
```

Harus muncul:

```text
MariaDB 11.4
```

---



# Ubah Password Root MariaDB untuk phpMyAdmin

Masuk MariaDB:

```bash
sudo mysql
```

Ubah password root:

```sql
ALTER USER 'root'@'localhost' IDENTIFIED VIA mysql_native_password USING PASSWORD('phpmyadmin');
FLUSH PRIVILEGES;
EXIT;
```

Restart MariaDB:

```bash
sudo systemctl restart mariadb
```

# Install mkcert
sudo apt update
sudo apt install libnss3-tools
wget https://github.com/FiloSottile/mkcert/releases/latest/download/mkcert-v1.4.4-linux-amd64
chmod +x mkcert-v1.4.4-linux-amd64
sudo mv mkcert-v1.4.4-linux-amd64 /usr/local/bin/mkcert
mkcert -version

# Install Local CA (Jalankan mkcert sebagai user biasa)
mkcert -install
# Outputnya biasanya seperti:
    Created a new local CA
    The local CA is now installed in the system trust store!
    The local CA is now installed in the Firefox and/or Chrome/Chromium trust store (requires browser restart)!
# Mulai sekarang Chrome akan mempercayai semua sertifikat yang dibuat oleh mkcert.

# Tambahkan hostname
sudo nano /etc/hosts
# Tambahkan:
127.0.0.1  opensimka.local
# Simpan

# Buat folder sertifikat (Jalankan mkcert sebagai user biasa)
mkdir ~/.ssl && cd ~/.ssl
# Jika generate sertifikat biasa (Jalankan mkcert sebagai user biasa):
    mkcert opensimka.local localhost 127.0.0.1 ::1
# Nantinya akan muncul dua file baru, masukkan ke vhost nginx
    ssl_certificate     /home/ideapad/.ssl/phpmyadmin.local+3.pem;
    ssl_certificate_key /home/ideapad/.ssl/phpmyadmin.local+3-key.pem;

sudo nginx -t
sudo nginx -s reload

## Buat virtual host Nginx

```bash
sudo nano /etc/nginx/sites-enabled/phpmyadmin.local
```

Isi:

```nginx
server {
    listen 80;
    server_name phpmyadmin.local;

    root /var/www/phpmyadmin;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.5-fpm.sock;
    }
}
```

## Login

Buka:

```text
http://phpmyadmin.local
```

Login phpMyAdmin:

```text
user: root
password: phpmyadmin
```


---

```bash
sudo nano /etc/phpmyadmin/config.inc.php
```

Tambahkan bagian ini di atas "phpMyAdmin configuration storage settings":

```
$cfg['Servers'][$i]['AllowNoPassword'] = false;
$cfg['Servers'][$i]['hide_db'] = 'mysql|information_schema|performance_schema|test|phpmyadmin|sys';
```

Tambahkan bagian ini sebelum "End of servers configuration":

```
$cfg['DefaultCharset'] = 'utf8';
$cfg['DefaultConnectionCollation'] = 'utf8mb4_general_ci';
$cfg['MaxNavigationItems'] = 1000;
$cfg['MaxTableList'] = 1000;
$cfg['NavigationTreeEnableGrouping'] = false;
$cfg['NavigationDisplayLogo'] = false;
```

Simpan.

Restart:

```bash
sudo systemctl restart nginx
sudo systemctl restart php8.5-fpm
```

###################################
# export no gzip
	date=$(date +%Y-%m-%d_%H-%M) && mariadb-dump -u root -p [DB_NAME] > /path/to/destination/[DB_NAME]_DB_${date}.sql
# export and gzip (-9 is highest compression)
	date=$(date +%Y-%m-%d_%H-%M) && mariadb-dump -u root -p [DB_NAME] | gzip -9 > /path/to/destination/[DB_NAME]_DB_${date}.sql.gz
###################################
######### Import Database #########
# make sure the database is EMPTY #
###################################
# import no extract
	mariadb -u [DB_USER] -p [DB_NAME] < file.sql
# import with extract
	gunzip < file.sql.gz | mariadb -u [DB_USER] -p [DB_NAME]
