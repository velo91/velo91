Installation If Dual Boot
======================================
just make sure Windows is "Disable Fast Boot" so that partition permissions on NTFS don't clash
1. type "control panel" into the search box on the taskbar and then click Control Panel in the search results
2. in the search box in the top right of the control panel, type power and then click Change what the power buttons do under Power Options
3. click "Change settings that are currently unavailable" and the give consent to continue or enter an administrator username and password as prompted
4. under Shutdown settings, uncheck "Turn on fast startup (recommended)" and click Save changes

SSH Key

# you can download ssh key from your google drive (never share it)
# or create a new one
$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
# press Enter to use the default location (~/.ssh/id_rsa)
# just leave blank the passphrase
# once done, an SSH key pair will be created and stored in the ~/.ssh/ directory:
	id_rsa: This is the private key (never share or show it).
	id_rsa.pub: This is the public key (this is what you can share with the server).
# add the public key to the server where you want to make a passwordless SSH connection
$ ssh-copy-id -p port_number username@hostname
# now you can try connecting to the server without a password
# all ssh connection stored in ~/.config/konsolesshconfig

	(…)

# with snap removed, you will need to install Firefox/Thunderbird from a PPA. Because apt will still try to install the Snap version of Firefox, you will run into an error. There's a trick to overcome that issue: pinning Firefox with Apt.
$ sudo nano /etc/apt/preferences.d/firefox-no-snap
	Package: firefox*
	Pin: release o=Ubuntu*
	Pin-Priority: -1
$ sudo nano /etc/apt/preferences.d/thunderbird-no-snap
	Package: thunderbird*
	Pin: release o=Ubuntu*
	Pin-Priority: -1
$ sudo add-apt-repository ppa:mozillateam/ppa && sudo apt install -y firefox thunderbird
# so, you should now have a snap-free Ubuntu with Firefox/Thunderbird installed.

# check has swap enabled, if output is empty, it means that your system does not have swap space enabled
$ swapon --show
# creating a file which will be used for swap
$ fallocate -l 4G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile && swapon --show
# make the change permanent
$ nano /etc/fstab
# paste it
/swapfile swap swap defaults 0 0
